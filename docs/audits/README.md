# Audits Index

## Ziel

Sammlung von Audit-Checklisten (Vorlagen) und angewendeten Audit-Reports für dieses Repository.

## Namensschema

- Checklisten (Vorlagen): `audit-checklist-<thema>.md`
- Angewendete Reports: `audit-report-<projekt>-YYYY-MM-DD.md`

Regeln:
- alles in `kebab-case`
- klarer Themenfokus im Dateinamen
- keine generischen Namen wie nur `readme.md`

## Checklisten (Vorlagen)

- `audit-checklist-ai-agent-linux-vm-enterprise-roocode-large-codebases.md`
- `audit-checklist-cybersecurity-ai-container-linux-vm-folder.md`
- `audit-checklist-git-workflow-devops.md`
- `audit-checklist-react-typescript-nextjs.md`
- `audit-checklist-readme-quality.md`

## Angewendete Reports

Aktuell keine dauerhafte Audit-Report-Datei im Repo.

Hinweis:
- Erkenntnisse aus der letzten Audit-Runde wurden direkt in `TODO.md` als priorisierte Aufgaben überführt.

## Nutzung

1. Passende Checkliste auswählen.
2. Scope festlegen (Repo-only, Runtime, Enterprise, etc.).
3. Ergebnisse entweder in einem separaten `audit-report-*` dokumentieren oder direkt als priorisierte Tasks in `TODO.md` überführen.
4. Findings priorisieren (`kritisch`, `mittel`, `nice-to-have`).
