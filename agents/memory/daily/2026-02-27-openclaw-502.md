# 2026-02-27 - Openclaw 502 Incident

## Kontext
- Externe URL `https://openclaw.tail027324.ts.net` lieferte `HTTP 502`.
- OpenClaw-Gateway (`127.0.0.1:18789`) lief, aber Bridge-Port `172.21.0.1:18790` war down.

## Ursache
- `tailscaled-openclaw` hatte eine veraltete `tailscale serve`-Route auf `http://172.21.0.1:18790`.
- Der Zielport war nicht erreichbar, weil der `socat`-Bridge-Prozess fehlte.

## Fix
```bash
HOST_GATEWAY_IP=$(docker network inspect ai-stack -f '{{(index .IPAM.Config 0).Gateway}}')
nohup socat TCP-LISTEN:18790,bind=${HOST_GATEWAY_IP},fork,reuseaddr TCP:127.0.0.1:18789 &
docker exec tailscaled-openclaw tailscale serve reset
docker exec tailscaled-openclaw tailscale serve --bg --https=443 "http://${HOST_GATEWAY_IP}:18790"
```

## Ergebnis
- `openclaw.tail027324.ts.net` liefert wieder OpenClaw (`HTTP 200`).
- Korrektur: Die zwischenzeitliche Umleitung auf MacroLens war ein falscher Workaround und wurde rückgängig gemacht.
